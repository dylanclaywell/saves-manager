## Added Windows install/update/run scripts for friend distribution

Mirroring the Linux deploy set (`install.sh`/`update.sh`/`savesmanager.service`) with a Windows-friendly equivalent. User isn't running on Windows themselves but wants to share with friends who would.

### Files added (all under `deploy/`)
- `run.ps1` / `run.bat` — start `node .output/server/index.mjs` in foreground from script-adjacent folder. `HOST=0.0.0.0 PORT=3000` defaults, overridable via `SAVESMANAGER_HOST`/`SAVESMANAGER_PORT` env vars. Bails with a friendly message if `.output/` or `node` is missing.
- `update.ps1` / `update.bat` — pulls `savesmanager-output.zip` from `releases/latest/download/`, extracts to temp, atomic swap into `.output/` (renames old to `.output.old`), cleans up temp. Configurable repo via `SAVESMANAGER_REPO`.
- `install.ps1` / `install.bat` — bootstrap. Checks for `node`; if missing AND `winget` available, prompts to install Node 20 LTS via `winget install OpenJS.NodeJS.LTS`. Otherwise points at nodejs.org. Then downloads run/update scripts from latest release into current folder and runs update.ps1 for the first build. install.bat is curl-bootstrappable: if install.ps1 isn't next to it, it fetches it from the release first.

### Why .bat wrappers
On stock Windows, double-clicking a `.ps1` either does nothing or triggers the execution-policy block. Each .bat is a one-liner that invokes powershell with `-NoProfile -ExecutionPolicy Bypass -File`. Friends double-click `install.bat`/`run.bat`/`update.bat` and never touch policy settings.

### .gitattributes updates
Critical: `.bat`/`.cmd` need CRLF (cmd.exe is finicky with LF), `.ps1` typically CRLF too. Previous rule was `deploy/** text eol=lf` which would have corrupted these. New rules:
- `*.sh`, `*.service`, `deploy/*.sh`, `deploy/*.service`, `.github/**` → LF
- `*.bat`, `*.cmd`, `*.ps1` → CRLF

### Workflow update
`release.yml` publish job's "Stage deploy assets" step now copies all 6 Windows scripts (plus the existing 4 Linux assets and the build zip) into `release-assets/` before `gh release upload --clobber`.

### Friend UX
1. Make a folder.
2. Download `install.bat` from latest release into that folder.
3. Double-click. Confirms Node install if missing (winget path), pulls scripts + first build.
4. Double-click `run.bat` to start the server. Browser to `http://localhost:3000`.
5. `update.bat` later to pull new releases.
